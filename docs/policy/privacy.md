# 개인정보 처리방침 초안

이 문서는 MVP 시연용 초안이며 법률 자문 문서가 아니다. 실제 서비스 출시 전에는 개인정보보호법, 의료/헬스케어 관련 규정, 위탁/제3자 제공 구조를 기준으로 법률 검토가 필요하다.

## 1. 수집하는 정보

서비스는 회원가입, 건강관리 기능 제공, 분석 결과 저장을 위해 아래 정보를 수집할 수 있다.

- 기본정보: 이름, 이메일, 로그인 ID, 비밀번호 해시, 성별, 생년월일, 휴대폰 번호
- 건강정보: 키, 체중, BMI, 혈압, 공복혈당, 콜레스테롤, 중성지방, HDL/LDL, 혈색소, 생활습관 정보
- 서비스 이용정보: 로그인 기록, 분석 요청 기록, 식단/챌린지/복약/알림 이용 기록
- 문의/고객지원 정보: 사용자가 입력한 문의 내용과 답변

주소는 1차 풀서비스 필수 수집 대상에서 제외한다. 지역 기반 병원/검진기관 추천, 배송, 방문 케어 기능이 도입될 경우 선택 정보로 재검토한다.

## 2. 이용 목적

수집한 정보는 아래 목적으로 사용한다.

- 회원 식별과 로그인/인증
- 간편 분석 및 정밀 분석 제공
- 식단, 챌린지, 복약/영양제, 알림 기능 제공
- 사용자 문의 대응
- 보안, 오류 추적, 서비스 품질 개선

분석 결과는 의료 진단, 확진, 치료, 처방이 아니라 건강관리 참고용 안내로 제공한다.

## 3. 민감정보와 건강정보

건강검진 수치, 식단, 복약 정보 등은 민감한 건강정보로 취급한다. 서비스는 기능 제공에 필요한 최소 범위로 건강정보를 수집하고, 사용자 화면과 관리자 화면에서 과도한 원문 노출을 피한다.

## 4. 보관 및 삭제

정보는 서비스 제공에 필요한 기간 동안 보관한다. 사용자는 계정 삭제 또는 개인정보 삭제를 요청할 수 있다. 법령상 보존 의무가 있는 정보가 있다면 해당 기간 동안 별도 보관 후 삭제한다.

MVP 시연 단계에서는 자동 보관기간 만료/파기 배치가 완성되지 않았으므로 운영 진입 전 retention 정책과 삭제 자동화를 별도로 구현해야 한다.

## 5. 제3자 제공 및 외부 처리

서비스는 원칙적으로 사용자의 개인정보와 건강정보를 제3자에게 제공하지 않는다. 외부 OCR, LLM, SMS, 이메일 provider를 사용하는 경우에는 처리 목적, 전송 항목, 보관 여부를 명확히 고지해야 한다.

## 6. 보안

서비스는 비밀번호를 해시로 저장하고, refresh token cookie에는 HttpOnly 속성을 사용한다. 운영 환경에서는 HTTPS, Secure cookie, 접근 로그, 관리자 권한 분리, 민감정보 접근 통제를 적용해야 한다.

## 7. 문의

개인정보 관련 문의와 삭제 요청은 서비스 운영 담당자를 통해 접수한다. 실제 운영 전에는 담당 부서, 이메일, 처리 기한을 명시해야 한다.
