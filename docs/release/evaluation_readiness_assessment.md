## 3. 배포 전 보완 우선순위

### 🔴 즉시(평가 직결, 점수 상승폭 큼)
1. **P95 성능 테스트 (5-1)** — locust/k6로 핵심 API(로그인, 분석 polling, 대시보드, 챗봇) P95 측정 → 결과를 `docs`에 표로 첨부. 현재 "미수행=1점" 위험. **이것만으로 5-1이 1→4~5로 상승.**
2. **챗봇 LLM 동기호출 비동기화 (5-1·5-5)** — `AsyncOpenAI` 또는 `asyncio.to_thread`로 이벤트루프 블로킹 제거(이전 LLM 감사 참조). 동시요청 지연/타임아웃 방지 → 성능테스트 결과도 안정화.
3. **BUG-007 마무리 / BUG-011 커밋** — 대시보드 챌린지 이름 매핑(백엔드 title 임베드), 혈압 merge 정책 변경 커밋(이전 QA 감사 참조). UI/UX(4-x)·안정성(5-3) 직결.

### 🟠 배포 전 권장(중간 비용·효과)
4. **재현성 문서 (3-3)** — `seed=42` 고정 사실 + 동일 입력 반복추론 일관성 테스트 결과를 1페이지로 정리. LLM 결정성은 룰엔진 우선·strict JSON으로 통제됨을 명시(필요 시 `temperature=0` 설정).
5. **화면 흐름도 (1-3)** — 핵심 플로우(가입→건강정보→분석→대시보드/챌린지) 와이어프레임/플로우차트 1장 추가 → 1-3이 3→4~5.
6. **배포 안정성 증빙 (5-3)** — 실 배포 후 `pre_deploy_checklist.md` 실행 로그/스크린샷, 헬스체크·기본 시나리오 통과 캡처. 모니터링(Sentry/헬스 알림) 연결.
7. **협업 증빙 (6-2)** — Issue/PR/리뷰/스프린트 보드 링크·캡처를 README나 별도 문서에 모아 5점 근거화.

### 🟡 여유 시(점수 보강)
8. 문제정의 정량 근거(만성질환 통계·타겟 규모) 한 단락(1-1).
9. 웹 기술스택 대안 비교 한 표(2-1).
10. 피드백→재학습 루프 설계 문구(3-4) — 수집된 confirm 데이터를 재학습 후보로 적재하는 계획 명시.
11. MVP 비노출 정리(관리자 링크/미사용 외부연동) — 이전 MVP 정리 보고서 반영(완성도 인상).
